Zepto é uma biblioteca JavaScript minimalista para navegadores modernos com uma API em grande parte compatível com jQuery. Se você usa jQuery, você já sabe como usar Zepto.

Enquanto 100% de cobertura jQuery não é um objetivo do projeto, as APIs fornecidas correspondem às suas contrapartes do jQuery. O objetivo é ter uma biblioteca modular ~ 5-10k que faz o download e executa rápido, com uma API familiar e versátil, para que você possa se ​​concentrar em obter o material feito.

Zepto é um biblioteca de código aberto e foi lançada sob a licença MIT.

Zepto só irá definir o $ global a si mesmo se ainda não está definido. Não existe um método Zepto.noConflict.

Se você precisa dá suporte para navegadores antigos, como o Internet Explorer 9 ou anterior, você pode utilizar o jQuery 1.x como fall back.

Navegadores suportados

Principal (100% suportado)

  • Safari 6+ (Mac)
  • Chrome 30+ (Windows, Mac, Android, iOS, Linux, Chrome OS)
  • Firefox 24+ (Windows, Mac, Android, Linux, Firefox OS)
  • iOS 5+ Safari
  • Android 2.3+ Browser
  • Internet Explorer 10+ (Windows, Windows Phone)

Alvos secundários (suportado totalmente ou em grande parte)

  • iOS 3+ Safari
  • Chrome <30
  • Firefox 4+
  • Safari <6
  • Navegador do Android 2.2
  • Opera 10+
  • Navegador webOS 1.4.5+
  • Navegador do tablet BlackBerry OS 1.0.7+
  • Amazon Silk 1.0+
  • Outros navegadores/plataformmas baseadas em WebKit

Note que alguns recursos opcionais do Zepto visam especificamente navegadores móveis, como o objetivo inicial do projeto era fornecer uma alternativa mais enxuta especificamente para jQuery para a web móvel.

Zepto é uma boa opção para criar extensões do navegador (para o Safari, Chrome e Firefox) e para desenvolver views baseadas em HTML em frameworks para aplicativos nativos, como o PhoneGap.

Em suma, Zepto é esperado que funcionem em todos os navegadores modernos. Zepto não tem suporte para versões antigas do Internet Explorer (<10).

Construindo manualmente o Zepto

zepto.js e zepto.min.js fornecidas acima podem ser usadas como está. No entanto, para melhor eficiência e personalização, execute o sistema de construção que vem com o código fonte do Zepto. Ele permite que você selecione os módulos, execute testes, use UglifyJS para minificar sua forma personalizada, e dá-lhe uma estimativa sobre a compressão que é viável quando zepto.min.js é servido gzippado.

Consulte o README para instruções sobre como personalizar o Zepto, incluindo a execução dos testes e contribuindo patches.

Zepto modules

module default description
zepto ✔ Core module; contains most methods
event ✔ Event handling via on() & off()
ajax ✔ XMLHttpRequest and JSONP functionality
form ✔ Serialize & submit web forms
ie ✔ Add support for Internet Explorer 10+ on desktop and Windows Phone 8.
detect Provides $.os and $.browser information
fx The animate() method
fx_methods Animated show, hide, toggle, and fade*() methods.
assets Experimental support for cleaning up iOS memory after removing image elements from the DOM.
data A full-blown data() method, capable of storing arbitrary objects in memory.
deferred Provides $.Deferred promises API. Depends on the "callbacks" module.
When included, $.ajax() supports a promise interface for chaining callbacks.
callbacks Provides $.Callbacks for use in "deferred" module.
selector Experimental jQuery CSS extensions support for functionality such as $('div:first') and el.is(':visible').
touch Fires tap– and swipe–related events on touch devices. This works with both `touch` (iOS, Android) and `pointer` events (Windows Phone).
gesture Fires pinch gesture events on touch devices
stack Provides andSelf & end() chaining methods
ios3 String.prototype.trim and Array.prototype.reduce methods (if they are missing) for compatibility with iOS 3.x.

Criando plug-ins

Plugins podem ser escritos adicionando métodos como propriedades de $.fn:

{% highlight js %} ;(function($){ $.extend($.fn, { foo: function(){ // `this` refers to the current Zepto collection. // When possible, return the Zepto collection to allow chaining. return this.html('bar') } }) })(Zepto) {% endhighlight %}

Para começar a desenvolver seu plug-in, dê uma olhada em código fonte do modulo central do Zepto's, e não se esqueça de ler as diretrizes de estilo de codificação.
